## 2. 原則一覧

### 2.1 原則サマリテーブル

| #   | 原則名    | 概要                                |
| --- | ------ | --------------------------------- |
| 1   | 不滅性    | 複雑性は消滅せず、別の領域に移転するのみである           |
| 2   | 自己言及性  | フレームワークの自己検証は原理的に不完全であり、自己矛盾を生成する |
| 3   | 観測起爆性  | 法則の内部構造を観測する行為自体が法則を発動させる         |
| 4   | 簡素化抵抗性 | 簡素化の試みは新たな複雑性を生成し、法則を強化する         |
| 5   | 無限後退性  | 競合処理の連鎖は原理的に終端しない                 |
| 6   | 分割増殖性  | モジュラー分割は管理複雑性を指数関数的に増大させ得る        |
| 7   | 文脈依存性  | 文脈なしの検証は構造的に不完全である                |
| 8   | 汎用自殺性  | 汎用性の拡大は汎用性自体を毀損する                 |

### 2.2 自己適用結果テーブル

| #   | 原則名    | 本法則自身に適用した結果                  |
| --- | ------ | ----------------------------- |
| 1   | 不滅性    | 本法則の定式化により複雑性が法則内部に移転している     |
| 2   | 自己言及性  | 本原則自身が自己言及構造を持ち矛盾を内包している      |
| 3   | 観測起爆性  | 本表を読解する行為が法則の作用を発動させている       |
| 4   | 簡素化抵抗性 | 原則の削減を試みると削減基準の定義が必要になる       |
| 5   | 無限後退性  | 原則間の矛盾処理が新たな原則を要求する           |
| 6   | 分割増殖性  | 8原則への分割が原則数の増大圧力を生んでいる        |
| 7   | 文脈依存性  | 本法則の適用文脈が定義されていない             |
| 8   | 汎用自殺性  | あらゆる場面への適用を主張するほど実用的有効性が毀損される |
